дј—жүҖе‘ЁзҹҘпјҢDubbo еҚҸи®®жҳҜзӣҙжҺҘе®ҡд№үеңЁ TCP дј иҫ“еұӮеҚҸи®®д№ӢдёҠпјҢз”ұдәҺ TCP й«ҳеҸҜйқ е…ЁеҸҢе·Ҙзҡ„зү№зӮ№пјҢдёә Dubbo еҚҸи®®зҡ„е®ҡд№үжҸҗдҫӣдәҶжңҖеӨ§зҡ„зҒөжҙ»жҖ§пјҢдҪҶеҗҢж—¶д№ҹжӯЈжҳҜеӣ дёәиҝҷж ·зҡ„зҒөжҙ»жҖ§пјҢRPC еҚҸи®®жҷ®йҒҚйғҪжҳҜе®ҡеҲ¶еҢ–зҡ„з§ҒжңүеҚҸи®®пјҢDubbo еҗҢж ·д№ҹйқўдёҙиҝҷдёӘй—®йўҳгҖӮеңЁиҝҷйҮҢжҲ‘д»¬зқҖйҮҚи®ІдёҖдёӢ Dubbo еңЁеҚҸи®®йҖҡз”ЁжҖ§ж–№йқўеҖјеҫ—ж”№иҝӣзҡ„ең°ж–№пјҢе…ідәҺеҚҸи®®иҜҰз»Ҷи§ЈжһҗиҜ·еҸӮи§Ғе®ҳзҪ‘еҚҡе®ў

  • Dubbo еҚҸи®®дҪ“ Body дёӯжңүдёҖдёӘеҸҜжү©еұ•зҡ„ attachments йғЁеҲҶпјҢиҝҷз»ҷ RPC ж–№жі•д№ӢеӨ–йўқеӨ–дј йҖ’йҷ„еҠ еұһжҖ§жҸҗдҫӣдәҶеҸҜиғҪпјҢжҳҜдёҖдёӘеҫҲеҘҪзҡ„и®ҫи®ЎгҖӮдҪҶжҳҜзұ»дјјзҡ„ Header йғЁеҲҶпјҢеҚҙзјәе°‘зұ»дјјзҡ„еҸҜжү©еұ• attachmentsпјҢиҝҷзӮ№еҸҜеҸӮиҖғ HTTP е®ҡд№үзҡ„ Ascii Header и®ҫи®ЎпјҢе°Ҷ Body Attachments е’Ң Header Attachments еҒҡиҒҢиҙЈеҲ’еҲҶгҖӮ
  • Body еҚҸи®®дҪ“дёӯзҡ„дёҖдәӣ RPC иҜ·жұӮе®ҡдҪҚз¬ҰеҰӮ Service NameгҖҒMethod NameгҖҒVersion зӯүпјҢеҸҜд»ҘжҸҗеҲ° Header дёӯпјҢе’Ңе…·дҪ“зҡ„еәҸеҲ—еҢ–еҚҸи®®и§ЈиҖҰпјҢд»ҘжӣҙеҘҪзҡ„иў«зҪ‘з»ңеҹәзЎҖи®ҫж–ҪиҜҶеҲ«жҲ–з”ЁдәҺжөҒйҮҸз®ЎжҺ§гҖӮ

зӣёжҜ”дәҺзӣҙжҺҘжһ„е»әдёҺ TPC дј иҫ“еұӮзҡ„з§Ғжңү RPC еҚҸи®®пјҢжһ„е»әдәҺ HTTP д№ӢдёҠзҡ„иҝңзЁӢи°ғз”Ёи§ЈеҶіж–№жЎҲдјҡжңүжӣҙеҘҪзҡ„йҖҡз”ЁжҖ§пјҢеҰӮWebServices жҲ– REST жһ¶жһ„пјҢдҪҝз”Ё HTTP + JSON еҸҜд»ҘиҜҙжҳҜдёҖдёӘдәӢе®һж ҮеҮҶзҡ„и§ЈеҶіж–№жЎҲгҖӮ

gRPC

дёҠйқўжҸҗеҲ°дәҶеңЁ HTTP еҸҠ TCP еҚҸи®®д№ӢдёҠжһ„е»ә RPC еҚҸи®®еҗ„иҮӘзҡ„дјҳзјәзӮ№пјҢзӣёжҜ”дәҺ Dubbo жһ„е»әдәҺ TPC дј иҫ“еұӮд№ӢдёҠпјҢGoogle йҖүжӢ©е°Ҷ gRPC зӣҙжҺҘе®ҡд№үеңЁ HTTP/2 еҚҸи®®д№ӢдёҠпјҢе…ідәҺ gRPC зҡ„ еҹәжң¬д»Ӣз»Қе’Ң и®ҫи®Ўж„ҝжҷҜ иҜ·еҸӮиҖғд»ҘдёҠдёӨзҜҮж–Үз« пјҢжҲ‘иҝҷйҮҢд»…ж‘ҳеҸ– и®ҫи®Ўж„ҝжҷҜ дёӯеҮ дёӘиғҪеҸҚжҳ  gRPC и®ҫи®Ўзӣ®зҡ„зү№жҖ§жқҘеҒҡз®ҖеҚ•иҜҙжҳҺгҖӮ

  • Coverage & SimplicityпјҢеҚҸи®®и®ҫи®Ўе’ҢжЎҶжһ¶е®һзҺ°иҰҒи¶іеӨҹйҖҡз”Ёе’Ңз®ҖеҚ•пјҢиғҪиҝҗиЎҢеңЁд»»дҪ•и®ҫеӨҮд№ӢдёҠпјҢз”ҡиҮідёҖдәӣиө„жәҗйҰ–е…Ҳзҡ„еҰӮ IoTгҖҒMobile зӯүи®ҫеӨҮгҖӮ

  • Interoperability & ReachпјҢиҰҒжһ„е»әеңЁжӣҙйҖҡз”Ёзҡ„еҚҸи®®д№ӢдёҠпјҢеҚҸи®®жң¬иә«иҰҒиғҪиў«зҪ‘з»ңдёҠеҮ д№ҺжүҖжңүзҡ„еҹәзЎҖи®ҫж–ҪжүҖж”ҜжҢҒгҖӮ

  • General Purpose & PerformantпјҢиҰҒеңЁеңәжҷҜе’ҢжҖ§иғҪй—ҙеҒҡеҘҪе№іиЎЎпјҢйҰ–е…ҲеҚҸи®®жң¬иә«иҰҒжҳҜйҖӮз”ЁдәҺеҗ„з§ҚеңәжҷҜзҡ„пјҢеҗҢж—¶д№ҹиҰҒе°ҪйҮҸжңүй«ҳзҡ„жҖ§иғҪгҖӮ

  • Payload AgnosticпјҢеҚҸи®®дёҠдј иҫ“зҡ„иҙҹиҪҪиҰҒдҝқжҢҒиҜӯиЁҖе’Ңе№іеҸ°дёӯз«ӢгҖӮ

  • StreamingпјҢиҰҒж”ҜжҢҒ Request - ResponseгҖҒRequest - StreamгҖҒBi-Steam зӯүйҖҡдҝЎжЁЎеһӢгҖӮ

  • Flow ControlпјҢеҚҸи®®иҮӘиә«е…·еӨҮжөҒйҮҸж„ҹзҹҘе’ҢйҷҗеҲ¶зҡ„иғҪеҠӣгҖӮ

  • Metadata ExchangeпјҢеңЁ RPC жңҚеҠЎе®ҡд№үд№ӢеӨ–пјҢжҸҗдҫӣйўқеӨ–йҷ„еҠ ж•°жҚ®дј иҫ“зҡ„иғҪеҠӣгҖӮ

жҖ»зҡ„жқҘиҜҙпјҢеңЁиҝҷж ·зҡ„и®ҫи®ЎзҗҶеҝөжҢҮеҜјдёӢпјҢgRPC жңҖз»Ҳиў«и®ҫи®ЎдёәдёҖдёӘи·ЁиҜӯиЁҖгҖҒи·Ёе№іеҸ°зҡ„гҖҒйҖҡз”Ёзҡ„гҖҒй«ҳжҖ§иғҪзҡ„гҖҒеҹәдәҺ HTTP/2 зҡ„ RPC еҚҸи®®е’ҢжЎҶжһ¶гҖӮ

и·ЁиҜӯиЁҖзҡ„жңҚеҠЎејҖеҸ‘ж¶үеҸҠеҲ°еӨҡдёӘж–№йқўпјҢд»ҺжңҚеҠЎе®ҡд№үгҖҒRPC еҚҸи®®еҲ°еәҸеҲ—еҢ–еҚҸи®®йғҪиҰҒеҒҡеҲ°иҜӯиЁҖдёӯз«ӢпјҢеҗҢж—¶иҝҳй’ҲеҜ№жҜҸз§ҚиҜӯиЁҖжңүеҜ№еә”зҡ„ SDK е®һзҺ°гҖӮиҷҪз„¶еҫ—зӣҠдәҺзӨҫеҢәзҡ„иҙЎзҢ®пјҢзҺ°еңЁ Dubbo еңЁеӨҡиҜӯиЁҖ SDK е®һзҺ°дёҠйҖҗжӯҘжңүдәҶиө·иүІпјҢе·Із»ҸжҸҗдҫӣдәҶеҢ…жӢ¬ Java, Go, PHP, C#, Python, NodeJs, C зӯүзүҲжң¬зҡ„е®ўжҲ·з«ҜжҲ–е…ЁйҮҸе®һзҺ°зүҲжң¬пјҢдҪҶеңЁд»ҘдёҠжҸҗеҲ°зҡ„и·ЁиҜӯиЁҖеҸӢеҘҪеһӢж–№йқўпјҢд»ҘдёҠдёүзӮ№иҝҳжҳҜжңүеҫҲеӨҡеҸҜж”№иҝӣд№ӢеӨ„гҖӮ

  • еҚҸи®®пјҢдёҠйқўжҲ‘д»¬е·Із»ҸеҲҶжһҗиҝҮ Dubbo еҚҸи®®ж—ўжңүзҡ„зјәзӮ№пјҢеҰӮжһңиғҪеңЁ HTTP/2 д№ӢдёҠжһ„е»әеә”з”ЁеұӮеҚҸи®®пјҢеҲҷж— з–‘иғҪйҒҝе…ҚиҝҷдәӣејҠз«ҜпјҢеҗҢж—¶жңҖеӨ§еҸҜиғҪзҡ„жҸҗй«ҳеҚҸи®®зҡ„з©ҝйҖҸжҖ§пјҢйҒҝе…ҚзҪ‘е…ізӯүеҚҸи®®иҪ¬жҚўз»„д»¶зҡ„еӯҳеңЁпјҢжӣҙжңүеҲ©дәҺй“ҫи·ҜдёҠзҡ„жөҒйҮҸз®ЎжҺ§гҖӮиҖғиҷ‘еҲ° gRPC жҳҜжһ„е»әеңЁ HTTP/2 д№ӢдёҠпјҢе№¶дё”е·Із»ҸжҳҜдә‘еҺҹз”ҹйўҶеҹҹжҺЁиҚҗзҡ„йҖҡдҝЎеҚҸи®®пјҢDubbo еңЁз¬¬дёҖйҳ¶ж®өйҖүжӢ©дәҶзӣҙжҺҘж”ҜжҢҒ gRPC еҚҸи®®дҪңдёәеҪ“еүҚзҡ„ HTTP/2 и§ЈеҶіж–№жЎҲгҖӮжҲ‘д»¬д№ҹзҹҘйҒ“ gRPC жЎҶжһ¶иҮӘиә«зҡ„ејҠз«ҜеңЁдәҺжҳ“з”ЁжҖ§дёҚи¶ід»ҘеҸҠжңҚеҠЎжІ»зҗҶиғҪеҠӣж¬ зјәпјҲиҝҷд№ҹжҳҜзӣ®еүҚз»қеӨ§еӨҡж•°еҺӮе•ҶдёҚдјҡзӣҙжҺҘиЈёз”Ё gRPC жЎҶжһ¶зҡ„еҺҹеӣ пјүпјҢйҖҡиҝҮе°Ҷе…¶йӣҶжҲҗиҝӣ Dubbo жЎҶжһ¶пјҢз”ЁжҲ·еҸҜд»Ҙж–№дҫҝзҡ„дҪҝз”Ё Dubbo зј–зЁӢжЁЎеһӢ + Dubbo жңҚеҠЎжІ»зҗҶ + gRPC еҚҸи®®йҖҡдҝЎзҡ„з»„еҗҲгҖӮ
